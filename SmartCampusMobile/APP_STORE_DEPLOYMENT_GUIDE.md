## Initial Setup

### 1. Initialize EAS Project

```bash
# Initialize EAS in your project
eas init

# This will:
# - Create an EAS project
# - Link it to your Expo account
# - Generate a project ID
# - Update app.json with the project ID
```

### 2. Configure Credentials

#### For iOS:

```bash
# Generate iOS credentials (certificates, provisioning profiles)
eas credentials

# Follow prompts to:
# - Create Apple Distribution Certificate
# - Create Provisioning Profile
# - Link to your Apple Developer account
```

#### For Android:

```bash
# Generate Android keystore
eas credentials

# Follow prompts to:
# - Create upload keystore
# - Set keystore password
# - Set key alias and password
```

### 3. Update Configuration

Update `app.json` with your specific details:

```json
{
  "expo": {
    "name": "Smart Campus",
    "slug": "smart-campus",
    "ios": {
      "bundleIdentifier": "com.yourcompany.smartcampus"  // Update this
    },
    "android": {
      "package": "com.yourcompany.smartcampus"  // Update this
    },
    "owner": "your-expo-username",  // Update this
    "extra": {
      "eas": {
        "projectId": "your-eas-project-id"  // Auto-generated by eas init
      }
    }
  }
}
```

## iOS App Store Deployment

### Step 1: Apple Developer Account Setup

1. **Enroll in Apple Developer Program**
   - Visit [developer.apple.com/programs](https://developer.apple.com/programs)
   - Complete enrollment ($99/year)
   - Wait for approval (usually 24-48 hours)

2. **Create App Store Connect App**
   - Go to [App Store Connect](https://appstoreconnect.apple.com)
   - Click "My Apps" → "+" → "New App"
   - Fill in:
     - Platform: iOS
     - Name: Smart Campus
     - Primary Language: English
     - Bundle ID: `com.smartcampus.mobile` (must match app.json)
     - SKU: `smart-campus-001` (unique identifier)
     - User Access: Full Access

3. **Prepare App Information**
   - App Category: Education
   - Content Rights: Check if applicable
   - Age Rating: Complete questionnaire
   - App Privacy: Add privacy policy URL

### Step 2: Create App Assets

#### Screenshots Required:
- **iPhone 6.7" (iPhone 15 Pro Max)**: 1290 x 2796 px
- **iPhone 6.5" (iPhone XS Max)**: 1242 x 2688 px
- **iPad Pro 12.9" (3rd gen)**: 2048 x 2732 px

#### App Icon:
- **1024 x 1024 px** (no transparency, PNG)

#### App Preview Videos (Optional):
- Up to 3 videos per device size
- 15-30 seconds recommended

### Step 3: Build for iOS

```bash
# Build for iOS App Store
eas build --platform ios --profile production

# This will:
# - Bundle your app
# - Sign with certificates
# - Upload to Expo servers
# - Generate .ipa file
```

### Step 4: Submit to App Store

```bash
# Automated submission
eas submit --platform ios --latest

# Or manually:
# 1. Download .ipa from Expo dashboard
# 2. Use Transporter app to upload to App Store Connect
# 3. Go to App Store Connect and submit for review
```

### Step 5: App Store Review Preparation

**Test Account Credentials** (for Apple reviewers):
```

Password: ReviewPass123!
School ID: SCH001
```

**App Review Information**:
- First Name: Smart
- Last Name: Campus

- Email: support@smartcampus.com
- Demo Account: Include above credentials
- Notes: "School management app requiring school credentials"

## Google Play Store Deployment

### Step 1: Google Play Console Setup

1. **Create Developer Account**
   - Visit [Google Play Console](https://play.google.com/console)
   - Pay $25 one-time registration fee
   - Complete account verification

2. **Create New App**
   - Click "Create app"
   - Fill in:
     - App name: Smart Campus
     - Default language: English
     - App or game: App
     - Free or paid: Free
   - Accept declarations

3. **Complete Store Listing**

   **App Details**:
   - Short description (80 chars max)
   - Full description (4000 chars max)
   - App icon: 512 x 512 px (PNG, 32-bit)
   - Feature graphic: 1024 x 500 px

   **Screenshots**:
   - Phone: At least 2 (JPEG/PNG, 16:9 or 9:16)
   - 7-inch tablet: At least 2
   - 10-inch tablet: At least 2

   **Categorization**:
   - App category: Education
   - Tags: School, Education, Management

   **Contact Details**:
   - Email: support@smartcampus.com
   - Website: https://smartcampus.com
   - Privacy policy: [Your privacy policy URL]

### Step 2: Set Up Release

1. **Create Service Account** (for automated submission)
   ```bash
   # Go to Google Cloud Console
   # Enable Google Play Developer API
   # Create service account
   # Download JSON key file
   # Save as: android/service-account-key.json
   ```

2. **Configure Release Settings**
   - Go to "Production" → "Create new release"
   - App signing: Let Google manage (recommended)
   - Countries: Select target countries

### Step 3: Build for Android

```bash
# Build Android App Bundle (AAB)
eas build --platform android --profile production

# This generates an optimized .aab file
```

### Step 4: Submit to Google Play

```bash
# Automated submission
eas submit --platform android --latest

# Or manually:
# 1. Download .aab from Expo dashboard
# 2. Go to Play Console → Production → Create new release
# 3. Upload .aab file
# 4. Complete release details
# 5. Review and roll out
```

### Step 5: Content Rating

1. Go to "Content rating" in Play Console
2. Complete questionnaire:
   - App category: Education
   - Contains ads: No (or Yes if applicable)
   - User-generated content: No
3. Receive rating (usually Everyone)

### Step 6: Data Safety

Fill out Data Safety form:
- Data collection: Yes (email, name, school info)
- Data sharing: No (unless applicable)
- Security practices: Encryption in transit
- Privacy policy: Required

## Building with EAS

### Build Profiles

The `eas.json` file defines three build profiles:

#### 1. Development Build
```bash
eas build --platform android --profile development
```
- For testing on physical devices
- Includes dev tools
- Not for store submission

#### 2. Preview Build
```bash
eas build --platform ios --profile preview
```
- Internal testing
- Ad-hoc distribution (iOS)
- APK distribution (Android)

#### 3. Production Build
```bash
# iOS
eas build --platform ios --profile production

# Android
eas build --platform android --profile production

# Both platforms
eas build --platform all --profile production
```

### Build Commands Reference

```bash
# Check build status
eas build:list

# View specific build
eas build:view [BUILD_ID]

# Cancel a build
eas build:cancel [BUILD_ID]

# Download build artifact
eas build:download [BUILD_ID]

# Configure credentials
eas credentials

# Update app version
eas build:version:set
```

## Submission

### Automated Submission

Update `eas.json` submission config:

```json
{
  "submit": {
    "production": {
      "ios": {
        "appleId": "your-apple-id@email.com",
        "ascAppId": "1234567890",  // From App Store Connect
        "appleTeamId": "ABCDE12345"  // From Apple Developer
      },
      "android": {
        "serviceAccountKeyPath": "./android/service-account-key.json",
        "track": "production"  // or "beta", "alpha", "internal"
      }
    }
  }
}
```

### Submit Commands

```bash
# iOS Submission
eas submit --platform ios --latest

# Android Submission
eas submit --platform android --latest

# Both platforms
eas submit --platform all --latest

# Specific build
eas submit --platform ios --id [BUILD_ID]
```

## Post-Deployment

### 1. Monitor Reviews
- **iOS**: App Store Connect → Ratings and Reviews
- **Android**: Play Console → Ratings and Reviews
- Respond to reviews promptly

### 2. Track Analytics
- **iOS**: App Store Connect → Analytics
- **Android**: Play Console → Statistics
- Monitor downloads, active users, crashes

### 3. Handle Updates

#### Version Updates
```bash
# Update version in app.json
{
  "version": "1.0.1",  // Increment this
  "ios": {
    "buildNumber": "2"  // Increment this
  },
  "android": {
    "versionCode": 2  // Increment this
  }
}

# Build new version
eas build --platform all --profile production

# Submit update
eas submit --platform all --latest
```

#### Over-the-Air (OTA) Updates
```bash
# Publish OTA update (for minor changes)
eas update --branch production --message "Bug fixes and improvements"
```

### 4. Set Up TestFlight (iOS)
```bash
# Build for TestFlight
eas build --platform ios --profile production

# Submit to TestFlight
eas submit --platform ios --latest

# Invite beta testers in App Store Connect
```

### 5. Set Up Internal Testing (Android)
```bash
# Submit to internal testing track
eas submit --platform android --latest --track internal

# Manage testers in Play Console
```

## Troubleshooting

### Common iOS Issues

**Issue: Build fails with "No valid code signing identity"**
```bash
# Solution: Regenerate credentials
eas credentials
# Select: iOS → Production → Remove and create new
```

**Issue: App rejected for missing privacy policy**
```bash
# Solution: Add privacy policy URL in App Store Connect
# Also add to app.json if required
```

**Issue: Invalid bundle identifier**
```bash
# Solution: Ensure bundle ID in app.json matches App Store Connect
# Bundle ID format: com.company.appname (lowercase, no special chars)
```

### Common Android Issues

**Issue: Upload failed with "Version code already exists"**
```bash
# Solution: Increment versionCode in app.json
"android": {
  "versionCode": 2  // Increment this
}
```

**Issue: Missing app icon**
```bash
# Solution: Ensure adaptiveIcon is correctly configured
"android": {
  "adaptiveIcon": {
    "foregroundImage": "./assets/adaptive-icon.png",
    "backgroundColor": "#ffffff"
  }
}
```

**Issue: Build fails with keystore error**
```bash
# Solution: Reset keystore
eas credentials
# Select: Android → Production → Remove and create new keystore
```

### General Issues

**Issue: Build takes too long**
```bash
# Solution: Use priority builds (paid Expo plans)
# Or build during off-peak hours
```

**Issue: App crashes on startup**
```bash
# Solution: Check logs
eas build:view [BUILD_ID] --logs

# Test with development build first
eas build --platform android --profile development
```
